## விரைவு தொடக்கம்

இதை உங்கள் டெர்மினலில் ஒட்டவும்:

```bash
npm install -g omnimap && omm setup
```

பின்னர் உங்கள் AI குறியீட்டு கருவியைத் திறந்து `/omm-scan` திறனைப் பயன்படுத்தவும்:

```
/omm-scan
```

இது உங்கள் குறியீட்டுத் தளத்தை ஸ்கேன் செய்து, கட்டிடக்கலை ஆவணங்களை உருவாக்கி, தர மதிப்பெண் எட்டப்படும் வரை **தானாக மேம்படுத்தும்** (இயல்புநிலை 80).

அவ்வளவுதான். முடிவைக் காணவும்:

```bash
omm view
```

## இது எப்படி வேலை செய்கிறது

உங்கள் AI குறியீட்டுத் தளத்தை பகுப்பாய்வு செய்து **கண்ணோட்டங்களை** உருவாக்குகிறது — உங்கள் கட்டிடக்கலையின் வெவ்வேறு கண்ணோட்டங்கள் (கட்டமைப்பு, தரவு ஓட்டம், ஒருங்கிணைப்புகள்...). ஒவ்வொரு கண்ணோட்டமும் ஒரு Mermaid வரைபடம் மற்றும் ஆவணப் புலங்களைக் கொண்டுள்ளது.

`omm analyze` tree-sitter AST பாகுபாட்டைப் பயன்படுத்தி சார்பு வரைபடங்கள், பொது API மேற்பரப்புகள் மற்றும் தொகுதி எல்லைகளை பிரித்தெடுக்கிறது — AI க்கு சொற்பொருள் பகுப்பாய்வுக்கு முன் ஒரு தீர்மானகரமான கட்டமைப்பு நங்கூரத்தை அளிக்கிறது.

ஒவ்வொரு முனையும் **மீண்டும் மீண்டும் பகுப்பாய்வு** செய்யப்படுகிறது. சிக்கலான முனைகள் அவற்றின் சொந்த வரைபடங்களுடன் உள்ளமைக்கப்பட்ட குழந்தை கூறுகளாக மாறும். எளியவை இலைகளாக இருக்கும். கோப்பு முறைமை மரத்தை பிரதிபலிக்கிறது:

```
.omm/
├── overall-architecture/           ← கண்ணோட்டம்
│   ├── description.md
│   ├── diagram.mmd
│   ├── context.md
│   ├── main-process/               ← உள்ளமைக்கப்பட்ட கூறு
│   │   ├── description.md
│   │   ├── diagram.mmd
│   │   └── auth-service/           ← ஆழமான உள்ளமைவு
│   │       └── ...
│   └── renderer/
│       └── ...
├── data-flow/
└── external-integrations/
```

காட்டி கோப்பு முறைமையிலிருந்து உள்ளமைவை தானாகக் கண்டறிகிறது — குழந்தைகள் உள்ள கூறுகள் விரிவாக்கக்கூடிய குழுக்களாக ரெண்டர் செய்யப்படும், மற்றவை முனைகளாக.

ஒவ்வொரு கூறும் 7 புலங்கள் வரை கொண்டுள்ளது: `description`, `diagram`, `context`, `constraint`, `concern`, `todo`, `note`.

## CLI

### முக்கிய கட்டளைகள்

```bash
omm setup [platform|--list|--teardown]  AI கருவிகளுக்கான திறன்களைப் பதிவு/ஆய்வு செய்யவும்
omm init [--template <name>]            .omm/ஐ துவக்கவும் அல்லது வார்ப்புருவிலிருந்து ஸ்காஃபோல்டு செய்யவும்
omm update                             CLI + செருகுநிரல்களை சமீபத்திய பதிப்பிற்கு புதுப்பிக்கவும்
omm config language <code>             உள்ளடக்க மொழியை அமைக்கவும் (en, ko, ja, zh, tr, ta)
```

### படித்தல் & எழுதுதல்

```bash
omm list [--project <name>]             கண்ணோட்டங்களை பட்டியலிடவும் (arch repo-ஐ தானாகக் கண்டறியும்)
omm tree [perspective]                  கூறு மரத்தை அச்சிடவும்
omm tree --yaml [--compact]            கூறு மரத்தை YAML-ஆக அச்சிடவும்
omm show <element>                      ஒரு கூறுக்கான அனைத்து புலங்களையும் காட்டவும்
omm show <element> --type               கூறு வகையைக் காட்டவும் (perspective/leaf/group)
omm inspect <element> [--json]          விரிவான கூறு ஆய்வு (மதிப்பெண், புலங்கள், இணைப்புகள்)
omm inspect <element> --links           @ref இணைப்பு தீர்வைக் காட்டவும் (சுழற்சிகள், உடைந்தவை)
omm read <path> <field>                 ஒரு புலத்தை stdout-க்கு படிக்கவும்
omm write <path> <field> <text|->       ஒரு புலத்தை எழுதவும்
omm status                              அனைத்து கூறுகளின் கண்ணோட்டத்தைக் காட்டவும்
```

### சரிபார்ப்பு

```bash
omm validate [path]                     வரைபடத்தை தொடரியல் மற்றும் மரபுகளுக்காக சரிபார்க்கவும்
omm validate <path> --fix               சரிசெய்யக்கூடிய சிக்கல்களை தானாக சரிசெய்யவும்
omm validate --explain                  சரிபார்ப்பு விதிகளை ஆவணப்படுத்தவும்
omm validate --rules                    ஒரு வரி விதி பட்டியல்
omm validate --changed [--json]         மாற்றப்பட்ட கூறுகளை மட்டும் சரிபார்க்கவும் (CI க்கு)
omm diff <element>                      வரைபட வேறுபாட்டைக் காட்டவும்
omm refs <element>                      உள்வரும்/வெளிச்செல்லும் குறிப்புகளைக் காட்டவும்
omm ref-syntax                           @class-name மரபை ஆவணப்படுத்தவும்
omm diagram-refs <path> [--json]         வரைபடத்தில் @refs-ஐ பட்டியலிடவும்
```

### தரம்

```bash
omm eval [--json] [--threshold <score>] ஆவணத் தரத்தை மதிப்பிடவும்
omm eval --explain <element>            ஒரு கூறுக்கான மதிப்பெண் பிரிவைக் காட்டவும்
omm eval --explain <element> --json      நிரல்முறை மதிப்பெண் பிரிவு (JSON)
omm eval --suggest                       மேம்படுத்த வேண்டிய முதல் 10 கூறுகள்
```

```bash
                                       வரைபடத்தை SVG, PNG அல்லது HTML-ஆக ஏற்றுமதி செய்யவும்
```

### அதிகரிக்கும்

```bash
omm incremental [--json]                அதிகரிக்கும் /omm-scan புதுப்பிப்புகளைத் திட்டமிடவும்
omm incremental --mark <p> [--files ...] [--globs ...] [--replace] [--recursive]
                                       ஒரு கூறுக்கான மூல கண்காணிப்பை துவக்கவும்
omm incremental --explain <element>      கூறு ஏன் பழையது/புதியது/அறியப்படாதது என்பதைக் காட்டவும்
omm incremental --record <p> [full|incremental]
                                       தற்போதைய commit-ல் கூறு ஸ்கேன் செய்யப்பட்டதாகக் குறிக்கவும்
```

பழைய காரணங்கள்:

| காரணம் | பொருள் |
|---|---|
| `source_file` | கண்காணிக்கப்பட்ட மூல கோப்பு மாறியது |
| `source_glob` | கண்காணிக்கப்பட்ட glob-ல் பொருந்தும் கோப்பு மாறியது |
| `orphaned_source` | கண்காணிக்கப்பட்ட மூல கோப்பு இனி இல்லை |
| `glob_coverage_changed` | glob இப்போது வெவ்வேறு கோப்புகளுடன் பொருந்துகிறது |
| `no_source_tracking` | கூறுக்கு source_files/globs இல்லை; பெற்றோரிடமிருந்து பழைமையை பெறுகிறது |
| `source_file_mtime` | மூல கோப்பு கடைசி ஸ்கேனை விட சமீபத்தில் மாற்றப்பட்டது |

### கட்டிடக்கலை களஞ்சியம்

```bash
omm push [--to repo] [--commit]         .omm/ஐ கட்டிடக்கலை களஞ்சியத்திற்கு அனுப்பவும்
omm pull [--from repo] [--all]          .omm/ஐ கட்டிடக்கலை களஞ்சியத்திலிருந்து பெறவும்
omm arch init [--remote <url>]          கட்டிடக்கலை களஞ்சியத்தை துவக்கவும்
omm share                               arch repo URL-ஐ அச்சிடவும்
omm org list|switch|add|edit|remove    கட்டிடக்கலை களஞ்சியங்களை நிர்வகிக்கவும்
```

```bash
omm analyze --diagram                   இறக்குமதி வரைபடத்திலிருந்து Mermaid வரைபடத்தை தானாக உருவாக்கவும்
omm analyze --validate                  .omm/ ஆவணங்களை உண்மையான குறியீட்டு கட்டமைப்புடன் ஒப்பிடவும்
omm analyze --impact <file>             ஒரு கோப்புக்கான மாற்ற தாக்கத்தைக் காட்டவும்
omm analyze --extensions                ஆதரிக்கப்படும் கோப்பு நீட்டிப்புகளைக் காட்டவும்
```

ஆதரிக்கப்படும் மொழிகள்: JavaScript, TypeScript, Java, Kotlin, Scala, Python, Go, Rust.

`--format md` கட்டிடக்கலை நுண்ணறிவுகளை உள்ளடக்கியது:
- **வட்ட சார்பு கண்டறிதல்** — இறக்குமதி சுழற்சிகள்
- **மைய முனைகள்** — அதிக சார்புள்ள முனைகள் (God nodes)
- **சமூகங்கள்** — தானாகக் கண்டறியப்பட்ட தொகுதி கொத்துகள்
- **இணைப்பு ஹாட்ஸ்பாட்கள்** — அதிக fan-in உள்ள கோப்புகள்
- **பயன்படுத்தப்படாத ஏற்றுமதிகள்** — பயன்பாடற்ற பொது குறியீடுகள் (Dead exports)
- **அடுக்கு மீறல் கண்டறிதல்** — கட்டிடக்கலை அடுக்குகளைத் தவிர்க்கும் விளிம்புகள்
- **கட்டிடக்கலை தர மதிப்பெண்** — 0-100 கலவை மதிப்பெண் (Architecture fitness score)
- **சிக்கலான ஹாட்ஸ்பாட்கள்** — 50 வரிகளுக்கு மேல் வரையறைகள்
- **வழிகாட்டப்பட்ட சுற்றுப்பயணம்** — ஆன்போர்டிங்கிற்கான படிக்கும் வரிசை

### வினவல் & தேடல்

வினவல் வடிவங்கள்: `"X-யும் Y-யும் இணைப்பது எது"`, `"X-ஐ யார் இறக்குமதி செய்கிறார்கள்"`, `"சுழற்சிகள்"`, `"ஹாட்ஸ்பாட்கள்"`, `"பயன்படுத்தப்படாதவை"`

## ஓட்டங்கள்

ஓட்டங்கள் உங்கள் வரைபடங்கள் வழியாக அனிமேட் செய்யப்பட்ட பாதைகளை வரையறுக்கின்றன. ஒவ்வொரு ஓட்டமும் செயல்படுத்தப்படும் போது ஒன்றாக சிறப்பிக்கும் முனைகள் மற்றும் விளிம்புகளின் வரிசை.

### ஓட்டங்களை உருவாக்குதல்

```bash
# ஒரு கூறுக்கான ஓட்டங்களை பட்டியலிடவும்
omm flows overall-architecture

# ஒரு ஓட்டத்தைச் சேர்க்கவும் (stdin-லிருந்து YAML படிக்கிறது)
omm flows overall-architecture add "Request Path" <<'EOF'
name: Request Path
description: ஒரு கோரிக்கை கணினி வழியாக எப்படி பாய்கிறது
steps:
  - node: client
  - edge: client->gateway
  - node: gateway
  - edge: gateway->service
  - node: service
EOF

# ஒரு ஓட்டத்தை அகற்றவும்
omm flows overall-architecture remove "Request Path"
```

### ஓட்ட YAML வடிவம்

```yaml
name: Install
description: டெவலப்பர் CLI மூலம் திறன்களை நிறுவுகிறார்
steps:
  - node: user          # இந்த முனையை சிறப்பிக்கவும்
  - edge: user->cli     # இந்த விளிம்பை சிறப்பிக்கவும் (from->to)
  - node: cli
  - edge: cli->output
  - node: output
```

- `node` படிகள் வரைபடத்திலிருந்து முனை IDகளைக் குறிக்கின்றன
- `edge` படிகள் `from->to` ஐக் குறிக்கின்றன, இங்கு from/to முனை IDகள்
- படிகள் வரிசைப்படுத்தப்பட்டவை — அவை காட்சி அனிமேஷன் வரிசையை வரையறுக்கின்றன

### காட்டியில் ஓட்டங்களைப் பயன்படுத்துதல்

1. `omm view` திறக்கவும்
2. எந்த கூறையும் கிளிக் செய்யவும் — ஓட்ட சிப்கள் கீழே தோன்றும்
3. பாதையை அனிமேட் செய்ய ஒரு சிப்பைக் கிளிக் செய்யவும்
4. செயல்நீக்க மீண்டும் கிளிக் செய்யவும்

### தானியங்கி உருவாக்கம்

`/omm-scan` திறன் ஸ்கேனின் போது ஓட்டங்களை தானாக உருவாக்குகிறது. இது நுழைவு புள்ளிகளிலிருந்து முனை முனைகளுக்கு பாதைகளை கண்டறிந்து ஒவ்வொரு கண்ணோட்டத்திற்கும் 2-5 ஓட்டங்களை உருவாக்குகிறது.

## கருத்து சுழற்சி

omm பராமரிப்பாளருடன் பகிர்ந்து கொள்ள `.omm/` க்குள் ஒரு கருத்து அறிக்கையை உருவாக்கவும்:

```bash
omm feedback                              # .omm/feedback.md எழுதுகிறது
omm feedback --format json                # .omm/feedback.json எழுதுகிறது
omm feedback --include "your message"     # சுதந்திர வடிவ சூழலைச் சேர்க்கவும்
omm feedback --print                      # stdout-க்கு அச்சிடவும்
```

உருவாக்கப்பட்ட கோப்பு பின்வருவனவற்றைக் கொண்டுள்ளது:

- omm பதிப்பு + git commit/கிளை
- திட்ட நிலை (கூறு எண்ணிக்கை, மதிப்பீட்டு மதிப்பெண், கவரேஜ் அளவீடுகள்)
- குறைந்த மதிப்பெண் கூறுகள் (முதல் 10)
- மதிப்பீட்டிலிருந்து சமீபத்திய சிக்கல்கள்
- மேம்பாட்டு பரிந்துரைகள்
- உங்கள் செய்தி (`--include` பயன்படுத்தப்பட்டிருந்தால்)

கோப்பு `.omm/` இல் இருப்பதால் திட்டத்துடன் பயணிக்கிறது. omm-ஐ மேம்படுத்த பராமரிப்பாளருடன் பகிரவும்.

```bash
omm eval                          # தர அறிக்கை
omm eval --json                   # இயந்திர-படிக்கக்கூடிய வெளியீடு
omm eval --threshold 80           # மதிப்பெண் < 80 என்றால் exit 1 (CI/CD)
```

அறிக்கை பின்வருவனவற்றை உள்ளடக்கியது:

- **ஒட்டுமொத்த மதிப்பெண்** (0-100) — புலங்கள், வரைபடம், ஓட்டங்கள், குறிப்புகள், குழந்தைகளால் எடையிடப்படும்
- **புல கவரேஜ்** — அனைத்து கூறுகளிலும் நிரப்பப்பட்ட புலங்களின் %
- **வரைபட கவரேஜ்** — சரியான mermaid வரைபடங்கள் உள்ள %
- **ஓட்ட கவரேஜ்** — ஓட்ட வரையறைகள் உள்ள %
- **குறிப்பு ஒருமைப்பாடு** — குறுக்கு-குறிப்புகள் உள்ள %
- **சிக்கல்கள்** — ஒரு கூறுக்கான பிழைகள், எச்சரிக்கைகள் மற்றும் தகவல்
- **பரிந்துரைகள்** — குறைந்த மதிப்பெண் பகுதிகளுக்கான மேம்பாடுகள்
- **குறைந்த மதிப்பெண் கூறுகள்** — மோசமான-முதல் வரிசைப்படுத்தப்பட்டவை

CI/CD குழாய்களில் குறைந்தபட்ச தர வரம்பை அமல்படுத்த `--threshold` பயன்படுத்தவும்.

## வரைபட காட்சிகள்

காட்டியில் இரண்டு வரைபட காட்சிப்படுத்தல் முறைகள் உள்ளன:

### உறவு வரைபடம் (◈ பொத்தான்)

- dagre படிநிலை அமைப்பைப் பயன்படுத்துகிறது (இடமிருந்து வலம்)
- குறுக்கு-கண்ணோட்ட இணைப்புகளை மட்டும் காட்டுகிறது
- நிலையானது — வழிசெலுத்த முனைகளைக் கிளிக் செய்யவும்
- சிறந்தது: கண்ணோட்ட ஒன்றுடன் ஒன்று, ஆவணப்படுத்தல்

### நெட்வொர்க் வரைபடம் (⬡ பொத்தான்)

- D3 விசை-வழிகாட்டப்பட்ட அமைப்பைப் பயன்படுத்துகிறது
- அனைத்து கூறுகள் (கண்ணோட்டங்கள் + குழந்தைகள்) மற்றும் அவற்றின் இணைப்புகளைக் காட்டுகிறது
- ஊடாடும் — மறுசீரமைக்க முனைகளை இழுக்கவும், ஜூம்/பான்
- அனிமேட் செய்யப்பட்டது — முனைகள் இயல்பாக நிலையை அடையும்
- கண்ணோட்டத்தால் வண்ண-குறியிடப்பட்டது
- இணைப்பு எண்ணிக்கையால் அளவு-குறியிடப்பட்டது
- சிறந்தது: ஆய்வு, கொத்துகளைக் கண்டறிதல், விளக்கக்காட்சிகள்

இரண்டு காட்சிகளும் மிதக்கும் கட்டுப்பாடுகளில் உள்ள அவற்றின் பொத்தான்கள் மூலம் மாற்றப்படுகின்றன.

## HTML ஏற்றுமதி

ஊடாடும் SVG, ஓட்ட அனிமேஷன், இருண்ட பயன்முறை மற்றும் கிளிக் செய்யக்கூடிய விவர அட்டைகளுடன் எந்த கூறையும் **சுய-உள்ளடக்கிய HTML கோப்பாக** ஏற்றுமதி செய்யவும்.

```bash
# CLI
omm export command-surface --format html -o diagram.html

# காட்டி
# எந்த கூறையும் கிளிக் செய்யவும் → ↓ பொத்தான் → HTML
```

HTML கோப்பு:
- முழுமையாக சுய-உள்ளடக்கியது (dagre.js CDN தவிர வெளிப்புற சார்புகள் இல்லை)
- `prefers-color-scheme` கண்டறிதலுடன் இருண்ட/ஒளி கருப்பொருள் மாற்றத்தை உள்ளடக்கியது
- ஓட்ட அனிமேஷன் சிப்கள் உள்ளன (ஓட்டங்கள் வரையறுக்கப்பட்டிருந்தால்)
- முனை கிளிக்கில் விவர அட்டைகளைக் காட்டுகிறது
- முதல் ஏற்றத்திற்குப் பிறகு ஆஃப்லைனில் வேலை செய்கிறது

> **குறிப்பு:** `file://` URLகளிலிருந்து ஏற்றுமதி செய்யப்பட்ட HTML கோப்புகளைத் திறக்கும் போது, உலாவிகள் கன்சோலில் பாதுகாப்பு எச்சரிக்கையைக் காட்டலாம். இது தீங்கற்றது — வரைபடம் சரியாக ரெண்டர் செய்யப்படும்.

## திட்ட தேர்வி

arch repo-ல் ஒன்றுக்கு மேற்பட்ட திட்டங்கள் உள்ளபோது `omm view` இயக்கும்போது, குறிப்பிட்ட திட்டத்தை நேரடியாக ஏற்றுவதற்குப் பதிலாக `http://localhost:3000/` இல் ஒரு **திட்ட தேர்வி** தானாகத் தோன்றும்.

தேர்வி ஒவ்வொரு திட்டத்தையும் பட்டியலிடுகிறது:
- கண்ணோட்ட எண்ணிக்கை
- கூறு எண்ணிக்கை
- கடைசியாக புதுப்பிக்கப்பட்ட நேரமுத்திரை

எந்த திட்ட அட்டையையும் கிளிக் செய்து காட்டியில் திறக்கவும்.

## கிளவுட்

நீங்கள் கட்டிடக்கலை ஆவணங்களை ஒரு **அர்ப்பணிக்கப்பட்ட git களஞ்சியத்தில்** சேமிக்கலாம் — வெளிப்புற சேவை தேவையில்லை.

### அமைப்பு (ஒரு முறை)

```bash
# பகிரப்பட்ட arch repo-ஐ அமைக்கவும்
omm config arch-repo ~/ws/my-mm-docs

# git remote-ஐ அமைக்கவும்
omm config arch-remote git@github.com:rsrini7/my-mm-docs.git
```

### குழுக்கள் (பல arch repos)

```bash
# குழு repos-ஐ பதிவு செய்யவும்
omm org add team-alpha ~/ws/alpha-repo --remote git@github.com:team/alpha.git
omm org add team-beta ~/ws/beta-repo --remote git@github.com:team/beta.git

# அனைத்தையும் பட்டியலிடவும்
omm org list

# செயலில் உள்ள repo-ஐ மாற்றவும்
omm org switch team-beta

# திருத்தவும்
omm org edit team-alpha --remote git@github.com:team/new-alpha.git

# அகற்றவும்
omm org remove team-beta
```

### பகிர்

```bash
# arch repo-க்கான GitHub/GitLab URL-ஐ அச்சிடவும்
omm share
```

### அனுப்பு

```bash
# .omm/ஐ arch repo-க்கு நகலெடுக்கவும்
omm push

# நகல் + git commit + remote-க்கு push
omm push --commit -m "add auth-service docs"

# என்ன மாறும் என்பதை முன்னோட்டமிடவும்
omm push --dry-run
```

| கட்டளை | கோப்புகளை நகலெடு | Git commit | Git push |
|---------|-----------|------------|----------|
| `omm push` | ✓ | ✗ | ✗ |
| `omm push --commit` | ✓ | ✓ | ✗ |
| `omm push --commit-push` | ✓ | ✓ | ✓ |
| `omm push --dry-run` | முன்னோட்டம் | ✗ | ✗ |

### பெறு

```bash
# arch repo-லிருந்து தற்போதைய திட்டத்தைப் பெறவும்
omm pull

# அனைத்து திட்டங்களையும் பெறவும்
omm pull --all

# குறிப்பிட்ட திட்டத்தைப் பெறவும்
omm pull --project ArcClawInternal
```

### Arch repo-லிருந்து நேரடியாகக் காணவும்

```bash
cd ~/ws/my-mm-docs

# அனைத்து திட்டங்கள் மற்றும் அவற்றின் கண்ணோட்டங்களை பட்டியலிடவும்
omm list

# குறிப்பிட்ட திட்டத்தைக் காணவும்
omm view --project ArcClawInternal

# குறிப்பிட்ட திட்டத்திலிருந்து ஒரு கூறைக் காட்டவும்
omm show command-surface --project ArcClawInternal
```

### அர்ப்பணிக்கப்பட்ட repo கட்டமைப்பு

```
~/arch/team-architecture/       ← பகிரப்பட்ட git repo
├── .omm/
│   ├── project-a/              ← ஒவ்வொரு திட்டத்திற்கும் தானாக உருவாக்கப்படும்
│   │   ├── auth-service/
│   │   └── api-gateway/
│   ├── project-b/
│   │   └── storage/
│   └── project-c/
│       └── ...
└── .git/
```

ஒவ்வொரு `omm push` உள்ளூர் `.omm/ஐ arch repo-ல் திட்ட பெயரின் கீழ் ஒத்திசைக்கிறது. குழு உறுப்பினர்கள் சமீபத்திய ஆவணங்களைப் பெற `omm pull` இயக்குகிறார்கள்.
